## Findings
- **Does removing premium 80-300 improve total PnL?** Yes. Best no-premium target-3 variant `BASE_ATM100_SCORE5_NO_PREMIUM_SPREAD1P5_SPREADPCT010_DEPTH250` made `13,075` vs baseline `11,525`.
- **Does it improve or worsen worst-day PnL?** Worsens. Best no-premium worst day was `-3,725` vs baseline `-600`.
- **Does it increase trade count dangerously?** Yes. Best no-premium variant used `123` trades vs baseline `76`.
- **Which premium buckets actually make money?** Best aggregated bucket was `150-300` at `75,650`; worst was `300-500` at `-35,100`.
- **Is premium band acting as useful quality filter, harmful overfit, or redundant because spread/depth already capture quality?** It is still a useful quality filter unless spread/depth are tightened materially. Plain no-premium made `12,300`, adding spread_pct guard made `12,575`, and the stricter liquidity-only version made `13,075`.
- **Does dynamic day-wise premium band outperform fixed band?** No. Best dynamic variant `DYNAMIC_PREMIUM_ATM100_SCORE5_MIDBAND_SPREAD2_DEPTH250` made `5,300` vs baseline `11,525`.
- **Direct recommendation:** `keep 80-300` as the current edge candidate, because no target-3 premium variant beat the current baseline while preserving downside control, PF >= 1.3, and trade-count discipline.
- **Research follow-up:** `BASE_ATM100_SCORE5_NO_PREMIUM_SPREAD1P5_SPREADPCT010_DEPTH250` is the only no-premium target-3 variant worth further study, but it is still research-only because it worsens the bad day and inflates trade count.
- **Secondary target-7 check:** `NO_PREMIUM_TARGET7_CHECK` improved headline PnL, but it is not the primary validation path because the standard here is target=3 first.
- **Practical answer:** keep `80-300` for the current tradable candidate; do not remove premium entirely, do not switch to dynamic percentile bands, and do not widen to `100-500`. If you want one exploratory branch, test the stricter no-premium spread/depth version only in research.
- **Actual live system over the same dates:** `-50,425` on `182` trades, PF `0.70`.
